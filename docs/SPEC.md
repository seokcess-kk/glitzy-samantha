# MMI (Medical Marketing Intelligence) 기능 명세서

## 1. 프로젝트 비전

### 1.1 목적
**"어떤 마케팅이 매출로 이어지는가?"**

병원 마케팅의 다채널 유입부터 결제까지 전체 퍼널을 추적하여,
마케팅 투자 대비 실제 매출 기여도를 측정하는 통합 대시보드.

### 1.2 핵심 가치
| 가치 | 설명 |
|------|------|
| **광고 ROI 가시화** | 어떤 광고 캠페인이 실제 매출을 만드는지 추적 |
| **고객 여정 통합** | 유입 → 상담 → 결제까지 하나의 흐름으로 연결 |
| **채널별 비교** | Meta, Google, 전화, 카카오 등 채널별 전환율 비교 |
| **멀티테넌트** | 여러 병원 고객사를 독립적으로 관리 |

### 1.3 대상 사용자
| 역할 | 사용자 | 핵심 니즈 |
|------|--------|----------|
| **슈퍼어드민** | Glitzy (마케팅 에이전시) | 전체 병원 관리, 계정 생성, 성과 비교 |
| **실행사 담당자** | 마케팅 실행사 직원 | 배정된 병원 순위 모니터링, 메뉴 권한 제한 |
| **병원 관리자** | 병원 담당자 | 자사 데이터 조회, 고객 관리, 담당자 관리 |
| **병원 담당자** | 병원 실무자 | 예약/결제, 고객, 리드, 챗봇만 접근 |

## 2. 유입 경로 정의

### 2.1 지원 채널

```
┌─────────────────────────────────────────────────────────────┐
│                    유입 경로 (4채널)                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  │  광고 클릭   │  │  전화 문의   │  │ 카카오톡 채널│  │ 네이버 예약  │
│  │  (UTM 추적)  │  │ (수동 등록)  │  │ (연동 예정)  │  │ (연동 예정)  │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘
│         │                 │                 │                 │
│         ▼                 ▼                 ▼                 ▼
│    웹폼 제출         직원 등록        챗봇 연동         API 연동
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 채널별 데이터 수집

| 채널 | 수집 방법 | 추적 데이터 | 구현 상태 |
|------|----------|------------|----------|
| **Meta Ads** | UTM + Webhook | utm_source=meta, campaign_id | ✅ 구현됨 |
| **Google Ads** | UTM + Webhook | utm_source=google, gclid | ✅ 구현됨 |
| **TikTok Ads** | UTM + Webhook | utm_source=tiktok | ✅ 구현됨 |
| **전화 문의** | 직원 수동 입력 | source=phone, 담당자 메모 | ✅ 구현됨 |
| **카카오톡** | 채널 연동 API | source=kakao | ⏳ 예정 |
| **네이버 예약** | 네이버 API | source=naver | ⏳ 예정 |

### 2.3 UTM 파라미터 체계

| 파라미터 | 용도 | 예시 |
|---------|------|------|
| `utm_source` | 유입 매체 | meta, google, naver, kakao |
| `utm_medium` | 광고 유형 | cpc, display, social, organic |
| `utm_campaign` | 캠페인명 | spring_promo, summer_sale |
| `utm_content` | 광고 소재 | banner_v1, video_30s |
| `utm_term` | 검색 키워드 | (Google Ads 키워드) |

## 3. 고객 식별 체계

### 3.1 전화번호 기반 식별

**선택 이유:**
1. 병원 특성상 예약/상담에 전화번호 필수
2. 유입 경로와 무관하게 수집 가능
3. 동명이인 문제 해결
4. 기존 시스템과 호환

### 3.2 식별 흐름

```
┌─────────────────────────────────────────────────────────────┐
│ 고객 식별 프로세스                                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  [신규 유입]                                                 │
│      │                                                       │
│      ▼                                                       │
│  전화번호 입력                                               │
│      │                                                       │
│      ▼                                                       │
│  정규화: 01012345678 → 010-1234-5678                        │
│      │                                                       │
│      ▼                                                       │
│  DB 조회: customers.phone_number = ?                        │
│      │                                                       │
│      ├─ NOT FOUND → 신규 고객 생성                          │
│      │              ├─ first_source 기록                    │
│      │              └─ first_campaign_id 기록               │
│      │                                                       │
│      └─ FOUND → 기존 고객 연결                              │
│                 └─ leads에 추가 유입 기록                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 데이터 구조

```
customers 테이블
├── id (PK)
├── phone_number (UNIQUE, 정규화됨) ← 식별자
├── name
├── first_source (최초 유입 채널: meta/google/kakao/naver/phone)
├── first_campaign_id (최초 캠페인)
├── clinic_id (FK)
├── total_visits (누적 방문 횟수) [계산 필드]
├── total_payments (누적 결제 금액) [계산 필드]
├── grade (등급: 일반/실버/골드/VIP) [계산 필드]
└── created_at
```

### 3.4 예외 처리

| 상황 | 처리 방법 |
|------|----------|
| 전화번호 변경 | 관리자가 수동으로 고객 병합 |
| 가족 공유 번호 | 동일 번호 + 다른 이름 = 별도 고객 |
| 번호 없음 (외국인) | 이메일 또는 임시 ID 발급 |

## 4. 풀퍼널 (Full-Funnel) 정의

### 4.1 전환 단계

```
┌─────────────────────────────────────────────────────────────┐
│ 고객 여정 퍼널                                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────┐                                            │
│  │ 1. 인입     │  광고 클릭 / 전화 / 카카오 / 네이버        │
│  │   (Lead)    │  → leads 테이블 생성                       │
│  └──────┬──────┘                                            │
│         │                                                    │
│         ▼ 전환율: Lead → Booking                            │
│  ┌─────────────┐                                            │
│  │ 2. 예약     │  상담 예약 확정                             │
│  │  (Booking)  │  → bookings 테이블 생성 (status: confirmed)│
│  └──────┬──────┘                                            │
│         │                                                    │
│         ▼ 전환율: Booking → Visit                           │
│  ┌─────────────┐                                            │
│  │ 3. 방문     │  실제 병원 방문                             │
│  │   (Visit)   │  → bookings.status = visited               │
│  └──────┬──────┘                                            │
│         │                                                    │
│         ▼ 전환율: Visit → Consultation                      │
│  ┌─────────────┐                                            │
│  │ 4. 상담     │  상담 진행                                  │
│  │(Consultation│  → consultations 테이블 생성               │
│  └──────┬──────┘                                            │
│         │                                                    │
│         ▼ 전환율: Consultation → Treatment                  │
│  ┌─────────────┐                                            │
│  │ 5. 시술확정 │  시술 결정                                  │
│  │ (Treatment) │  → bookings.status = treatment_confirmed   │
│  └──────┬──────┘                                            │
│         │                                                    │
│         ▼ 전환율: Treatment → Payment                       │
│  ┌─────────────┐                                            │
│  │ 6. 결제     │  결제 완료                                  │
│  │  (Payment)  │  → payments 테이블 생성                    │
│  └─────────────┘                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 상태 정의

**예약 상태 (bookings.status):**
| 상태 | 영문 | 설명 |
|------|------|------|
| 예약확정 | `confirmed` | 상담 예약 완료 |
| 방문완료 | `visited` | 병원 방문 확인 |
| 노쇼 | `noshow` | 예약 불이행 |
| 취소 | `cancelled` | 예약 취소 |
| 시술확정 | `treatment_confirmed` | 시술 결정 |

**상담 상태 (consultations.status):**
| 상태 | 설명 |
|------|------|
| 예약완료 | 상담 예약 등록 |
| 방문완료 | 상담실 입장 |
| 노쇼 | 상담 불참 |
| 상담중 | 상담 진행 중 |
| 취소 | 상담 취소 |
| 시술확정 | 시술 결정 완료 |

## 5. KPI 계산 명세

### 5.1 핵심 지표

| 지표 | 계산식 | 단위 | 용도 |
|------|--------|------|------|
| **CPL** | 광고비 ÷ 리드수 | 원 | 리드 획득 비용 |
| **ROAS** | (결제액 ÷ 광고비) × 100 | % | 광고 투자 수익률 |
| **CAC** | 총광고비 ÷ 결제고객수 | 원 | 고객 획득 비용 |
| **ARPC** | 총결제액 ÷ 결제고객수 | 원 | 고객당 평균 매출 |
| **CVR** | 전환수 ÷ 유입수 × 100 | % | 단계별 전환율 |
| **LTV** | 고객 누적 결제액 | 원 | 고객 생애 가치 |

### 5.2 집계 기준

모든 KPI는 다음 기준으로 집계 가능:

| 기준 | 설명 | 예시 |
|------|------|------|
| **전체** | 전체 데이터 집계 | 이번 달 전체 ROAS |
| **채널별** | utm_source 기준 | Meta ROAS vs Google ROAS |
| **캠페인별** | utm_campaign 기준 | spring_promo CPL |
| **기간별** | 일/주/월/분기 | 주간 리드 추이 |
| **병원별** | clinic_id 기준 | A병원 vs B병원 성과 |

### 5.3 전환율 계산

```
퍼널 전환율 예시:

Lead → Booking 전환율
  = (예약된 리드 수 ÷ 전체 리드 수) × 100

Booking → Visit 전환율
  = (방문한 예약 수 ÷ 전체 예약 수) × 100

Visit → Payment 전환율
  = (결제한 방문자 수 ÷ 전체 방문자 수) × 100

전체 전환율 (End-to-End)
  = (결제 고객 수 ÷ 전체 리드 수) × 100
```

### 5.4 고객 등급 기준

| 등급 | 조건 | 혜택/분류 |
|------|------|----------|
| **VIP** | 총결제액 ≥ 500만원 OR 시술 ≥ 5회 | 최우선 관리 |
| **골드** | 총결제액 ≥ 200만원 OR 시술 ≥ 3회 | 우선 관리 |
| **실버** | 총결제액 ≥ 50만원 OR 시술 ≥ 1회 | 일반 관리 |
| **일반** | 그 외 | 잠재 고객 |

### 6.2 핵심 테이블 스키마

#### customers (고객 마스터)
```sql
CREATE TABLE customers (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  phone_number VARCHAR(20) NOT NULL,        -- 정규화된 전화번호 (식별자)
  name VARCHAR(100),
  first_source VARCHAR(50),                 -- 최초 유입 채널
  first_campaign_id VARCHAR(100),           -- 최초 캠페인
  created_at TIMESTAMP DEFAULT NOW(),

  UNIQUE(clinic_id, phone_number)           -- 병원별 고유
);
```

#### leads (리드/유입 기록)
```sql
CREATE TABLE leads (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  customer_id INTEGER REFERENCES customers(id),

  -- UTM 파라미터 (개별 필드)
  utm_source VARCHAR(50),                   -- 유입 매체
  utm_medium VARCHAR(50),                   -- 광고 유형
  utm_campaign VARCHAR(100),                -- 캠페인명
  utm_content VARCHAR(100),                 -- 광고 소재
  utm_term VARCHAR(100),                    -- 검색 키워드

  -- 기존 필드
  campaign_id VARCHAR(100),                 -- 레거시 (호환용)
  inflow_url TEXT,                          -- 전체 URL

  -- 챗봇
  chatbot_sent BOOLEAN DEFAULT FALSE,
  chatbot_sent_at TIMESTAMP,

  created_at TIMESTAMP DEFAULT NOW()
);
```

#### bookings (예약)
```sql
CREATE TABLE bookings (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  customer_id INTEGER REFERENCES customers(id),
  booking_datetime TIMESTAMP NOT NULL,
  status VARCHAR(30) DEFAULT 'confirmed',   -- confirmed/visited/noshow/cancelled/treatment_confirmed
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### consultations (상담)
```sql
CREATE TABLE consultations (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  customer_id INTEGER REFERENCES customers(id),
  consultation_date DATE,
  status VARCHAR(20),                       -- 예약완료/방문완료/노쇼/상담중/취소/시술확정
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### payments (결제)
```sql
CREATE TABLE payments (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  customer_id INTEGER REFERENCES customers(id),
  treatment_name VARCHAR(200),              -- 시술명
  payment_amount INTEGER NOT NULL,          -- 결제 금액
  payment_date DATE NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### ad_campaign_stats (광고 통계)
```sql
CREATE TABLE ad_campaign_stats (
  id SERIAL PRIMARY KEY,
  clinic_id INTEGER REFERENCES clinics(id),
  platform VARCHAR(20),                     -- google/meta/tiktok/youtube
  campaign_id VARCHAR(100),
  campaign_name VARCHAR(200),
  date DATE,
  impressions INTEGER DEFAULT 0,
  clicks INTEGER DEFAULT 0,
  spend_amount DECIMAL(12,2) DEFAULT 0,
  conversions INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),

  UNIQUE(clinic_id, platform, campaign_id, date)
);
```

### 6.3 인덱스 권장

```sql
-- 고객 검색 최적화
CREATE INDEX idx_customers_phone ON customers(phone_number);
CREATE INDEX idx_customers_clinic ON customers(clinic_id);

-- 리드 분석 최적화
CREATE INDEX idx_leads_utm ON leads(utm_source, utm_campaign);
CREATE INDEX idx_leads_created ON leads(created_at);
CREATE INDEX idx_leads_clinic ON leads(clinic_id);

-- 결제 분석 최적화
CREATE INDEX idx_payments_date ON payments(payment_date);
CREATE INDEX idx_payments_customer ON payments(customer_id);

-- 광고 통계 최적화
CREATE INDEX idx_ad_stats_date ON ad_campaign_stats(date);
CREATE INDEX idx_ad_stats_campaign ON ad_campaign_stats(campaign_id);
```

**목적**: 광고 랜딩페이지에서 고객 정보 + UTM 파라미터 자동 수집

```

                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 랜딩페이지 (MMI 내장 또는 외부)                              │

│  ┌─────────────────────────────────────┐                    │
│  │ [문의 폼]                            │                    │
│  │                                      │                    │
│  │ 이름: [_____________]                │                    │
│  │ 연락처: [_____________]              │                    │
│  │                                      │                    │
│  │ [상담 신청하기]                      │                    │
│  │                                      │                    │
│  │ * UTM 파라미터 자동 수집 (hidden)    │                    │
│  └─────────────────────────────────────┘                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
```
